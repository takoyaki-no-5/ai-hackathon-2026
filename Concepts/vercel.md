### なにか

- **ホスティング＋デプロイの平台**（常時立てる自前サーバーではない）
- 画面（Vite/React や Next）を載せて **公開URL** を出すのが主用途
- 「バックエンド全部」ではない。LLM 本体は OpenAI 等の外、Vercel は橋渡し側

### サーバーレス？

- **はい（関数部分）**: API が必要なら Serverless Functions / Next の Route で、リクエストのときだけ動く
- フロントの静的配信も Vercel 上。常時起動の VM を自分で管理しない、という意味でも近い

### このハッカソンでの位置

- 路線 A の **公開URL担当** → [[stack]]
- 強み: `git push` 相当でデプロイが速い
- 弱点: 差別化は「何を載せるか」側。Vercel 自体は普通の選択

### 自分用メモ

- アカウント・プロジェクトを事前作成
- 環境変数（LLM APIキー）を Vercel に置く（フロントにベタ書きしない）
