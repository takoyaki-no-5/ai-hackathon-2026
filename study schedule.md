### 方針（勝ちに行く）

- 目標: **ブロック1位 → 決勝進出**（予選は「動く＋尖る」が足りれば届く）
- 新卒でも勝ち筋は「手書きの実力」ではなく **AIを開発そのものに食い込ませる速度と、テーマの読み替え**
- 2025決勝の決め手も「AIで大量に作り切る＋新しい手法をすぐ取り込む」側だった
- 審査4点を全部取りにいく: テーマ / **動く** / ハッカー魂 / **AI活用度**

### 勝ちの設計図（当日の頭の中）

1. 最初15分: テーマを **字面どおり＋比喩・読み替え** の2案出す → 尖る方を選ぶ（普通チャットは負け筋）
2. 最初45分で **審査員が触れる最小デモを公開URL** にする
3. 残り: 「他チームがやらない一手」を1つだけ足す（多モーダル / リアルタイム / 自前GPU推論 / 最新API 等）
4. 最後30分: 機能追加禁止。デモ台本・失敗時フォールバックだけ

### ゴール（7/24）

- [ ] 3時間模試を **3回**（うち1回はペア）完走・毎回公開URL
- [ ] 「テーマ読み替え → 尖った一文コンセプト」を10分で出せる
- [ ] Cursor等で **画面・API・デプロイを自分で止めない**（相手待ち禁止ルール）
- [ ] 3分デモを録画して客観視（操作中に沈黙しない）
- [ ] 差別化用の「裏技スロット」を1つ用意（自宅GPU or 画像/音声 or tool use 等）

### スタック（主戦場＋切り札）

- 主戦場: Next.js or Vite+React / 主要LLM API / Vercel（提出URL）
- 開発武器: Cursor + 必要なら Claude/Codex（**コードも設計もAIに書かせる**）
- 切り札（どちらか1つを仕込む）:
  - 自宅GPU: 会場ノート → 自宅推論（公開URL必須・回線プランB）
  - or クラウド最新機能（画像理解・音声・リアルタイム・エージェント）を当日即接続できる状態

#### 当日 7/25 — 勝ち方

- 0–15分: 解釈合戦。普通案は捨てる。ただし **動く道がある尖り** だけ残す
- 15–60分: MVPをデプロイ。ここでURLがないチームは負けスタート
- 60–150分: 差別化スロットを1つだけ乗せる。AI活用が画面上で分かるようにする
- 150–180分: デモ動線。失敗時は事前結果 or 簡易モードへ切替
- 審査: 触らせる → 驚きを一言 → どうAIを使ったかを必ず言う

### やること / やらないこと

- やる: AIにコードを大量生成させて統合する訓練、テーマの再解釈、デモの完成度
- やらない: 新FW学習、ゼロからのモデル学習、盛りすぎ機能、デザイン完璧主義

### チームルール（勝ち用）

- 案で揉めたら **3分で多数決 or コイン**。議論で15分以上使ったら負け
- 片方は常に「動く枝」、もう片方は「尖る枝」。最後にマージ
- 相手のレビュー待ち禁止。PRというより **すぐ本線に載せる**
