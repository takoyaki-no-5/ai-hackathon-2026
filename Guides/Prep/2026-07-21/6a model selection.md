### #6a モデル選定・発想テク

枠: **19:45–20:00**（15分）。担当: **K**（F は横で見てよい）。

目的: 木 R2 の **0–15分**（読み替え）を本番と同じ手順で1回やる。詳細: [[Concepts/model selection|model selection]]

### 完了の定義

- [ ] 同じ仮テーマで **temperature 0.2 vs 1.0** を1回ずつ比較、差を **1行メモ**
- [ ] 「読み替え案を5つ」で件数指定し、案が5つ出た
- [ ] 木 R2 の 0–15分手順を **口頭で1回** 説明できる

### 手順

#### 1. 起動確認（開始前・2分）

```powershell
git pull
pnpm dev
```

月の詰まりメモ（R1）を1つ見る。

#### 2. 仮テーマを決める（1分）

R1 と同じでよい。1文で貼る。

#### 3. temperature 比較（7分）

**安定モード（0.2）** — Cursor チャット or API:

```
テーマ: 「＜仮テーマ＞」
このテーマの読み替え案を3つ。各タイトル1行のみ。
```

**発想モード（1.0）** — 同じプロンプトを再実行（Cursor の temperature 設定 or API の `temperature: 1.0`）。

メモ例:

```
0.2=堅いSaaS寄り / 1.0=子供・ゲーム・意外な比喩が出た
```

#### 4. 件数指定ブレスト（5分）

```
テーマ: 「＜仮テーマ＞」
審査員が驚く読み替え案をちょうど5つ。番号付き。尖りは画像1つ想定。
```

5つ出たら OK。3分で1つ選ぶ練習は **木 R2** で本番。

#### 5. 口頭リハ（1分）

F に「木の最初15分は、(1)テーマ貼る (2)5案 (3)3分で決める」と説明。
