### #14 テザリングで同じURL

枠: **22:20–22:40**（20分）。担当: **K + F**。

目的: 会場 Wi‑Fi 不安時の **回線B** で、同じ公開 URL が開けることを確認。

### 完了の定義

- [ ] スマホがテザリング経由でインターネットに出られる
- [ ] スマホ **Chrome** で公開 URL が Mac と同様に開ける
- [ ] 当日の回線切替手順を1行メモ

### 手順

#### K（Win）— 参考

会場では K=Win / F=Mac。今夜は **F の Mac + iPhone テザリング** が本線。

#### F（Mac）

1. iPhone: 設定 → インターネット共有（テザリング）ON
2. Mac: Wi‑Fi → iPhone を選択
3. Chrome で公開 URL を開く
4. Agent を1回操作（送信 or 画像）

#### 問題がある場合

| 症状 | 確認 |
|------|------|
| URL が開かない | テザリングが本当に繋がっているか |
| 遅い | 本番はテザリングBを最初から使う選択肢 |
| 証明書エラー | URL の typo、Vercel 側の問題 |

#### メモ例

```
回線B: F iPhone テザリング → Mac Chrome で URL OK。会場ではWi-Fi不安なら最初からB。
```
