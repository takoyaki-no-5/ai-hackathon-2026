### 生成の暴力 × Web（6型の解説）

方針: 見た目をAIに連続生成する演出メモ。  
**位置づけはサブ・任意**（落としても本体は動く）。本線は Agent＋画像尖り → [[Guides/slot ideas|slot ideas]]

共通のコツ:
- 全部自由生成は壊れる → **テンプレ＋差し替えスロット**（色・時間・コピー・数個のクラス名だけAIに任せる等）
- ストリーミング or 短いチャンクで当てる（3分審査向き）
- 壊れたら直前の安全CSSに戻す

---

### 1. CSSライブ生成

**何か:** AIにCSS（特に `@keyframes`・色・余白・transform）を書かせ、`<style>` に流し込んで画面をPVみたいに動かす。

**流れ:**
1. 入力（テーマ語・スライダー・審査員の一言）
2. LLMがCSS文字列を返す
3. 既存スタイルを差し替え／追記
4. 必要なら数秒おきに「次の演出」を再生成

**例:** 「孤独」→ 青く暗い背景＋ゆっくり浮かぶ文字。「祭り」→ 赤黄・激しい点滅と拡大。

**強み:** 実装が比較的軽い。待ちが短い。Webそのものが作品。  
**弱み:** 変なCSSでレイアウト崩壊。セーフリスト必須。

---

### 2. SVG／Canvas連打

**何か:** AIに「図形の設計図」（パス座標・円の配列・JSON）を出させ、毎フレーム／毎秒 Canvas や SVG に描き直す。

**流れ:**
1. 「粒子が集まる」「顔の輪郭」など意図を渡す
2. AIが `{ shapes: [...] }` のような構造化出力
3. 描画ループがそれを読んでアニメーション

**例:** テーマ「波」→ 正弦波の折れ線を連続更新。「都市」→ 矩形のスカイラインが伸び縮み。

**強み:** CSSより「絵が動く」感が強い。構造化出力と相性良い。  
**弱み:** JSONが壊れると何も出ない。バリデーション必須。描画コードは自分で持つ。

---

### 3. シェーダ一言生成

**何か:** 画面全体にかける視覚効果（ぼかし・グリッチ・色収差など）を、短いシェーダ／CSS filter としてAIに書かせる。

**流れ:**
1. 「壊れたテレビ風」「水面」など一言
2. AIが短い GLSL 断片 or `filter:` 列を返す
3. Canvas/WebGL か CSS に適用

**例:** 審査員が「ノイズ」と言う → 画面が砂嵐＋色ずれ。

**強み:** 一言で世界観が変わる。デモ映え。  
**弱み:** GLSLは壊れやすい・デバッグ地獄。**予選では CSS `filter` / 用意済みシェーダのパラメータだけAIに選ばせる**方が安全。

---

### 4. DOMを物語にする

**何か:** 映画の「次のシーン」を、AIが小さなHTMLのかたまりとして返し、フェードで切り替える。

**流れ:**
1. 今のシーンIDとテーマを渡す
2. AIが次のHTML断片（見出し・短い文・ボタン）を返す
3. クロスフェードで差し替え
4. ユーザー操作 or 自動で次へ

**例:** テーマ「嘘」→ シーン1「真実の鏡」→ シーン2「鏡が割れる」→ シーン3「裏のメッセージ」。

**強み:** ストーリー性・テーマ読み替えが載る。審査員が「先が気になる」。  
**弱み:** HTMLが壊れる／XSS注意（サニタイズ）。シーン長を短く（各3〜8秒）。

---

### 5. タイポ演出

**何か:** テーマから **キャッチコピー＋その文字の動き（CSS）** を同時生成。3秒で「作品の顔」を出す。

**流れ:**
1. テーマ語だけ渡す
2. AIが `{ title, subtitle, cssMotion }` を返す
3. 大きい文字が画面に載ってアニメ

**例:** 「Communication」→ 「伝わらない夜に。」＋一文字ずつフェードイン。

**強み:** 最速で世界観。実装が軽い。他の型の「オープニング」にも使える。  
**弱み:** コピーが平凡だと弱い。日本語の改行・フォント指定はテンプレ側で固定。

---

### 6. UIそのものを生成

**何か:** 審査員の一言で、画面上の部品構成（ボタン数・配置・モード）が組み変わる。

**流れ:**
1. 「もっとシンプルに」「ゲームにして」など
2. AIがUIスキーマ（JSON: 部品リスト）を返す
3. React等がスキーマからコンポーネントを描画（HTML生埋めより安全）

**例:** 「診断アプリにして」→ スライダーと結果カードが出現。「チャットにして」→ 入力欄に切り替わる。

**強み:** 「AIがプロダクトを変えた」感が強い。ハッカー魂・AI活用度向け。  
**弱み:** スキーマ設計が先に必要。自由HTML生成より **JSON→既知コンポーネント** が安全。当日ゼロからは重い。

---

### どう組み合わせるか

| 役割 | 向き |
|------|------|
| 最初の3秒 | **5 タイポ** |
| 本体の動き | **1 CSS** or **2 Canvas** |
| 物語で刺す | **4 DOMシーン** |
| 一言で豹変 | **3 フィルタ** or **6 UIスキーマ** |

サブとしての最小セット: **5（顔）＋ html/css ライブ**。本体が落ちても本命A/Bは動くこと。

デモ（型1+4寄せ）: AIが `{ bg, html, css }`（`@keyframes`）を返す。

動く最小デモ:
1. `secrets/.env` にキー（雛形は `.env.example`）
2. `prototype/css-live` で `node server.mjs`
3. http://localhost:8787 → 「AIで表現」／「3シーン連続」
4. script / `url(` / `@import` はクライアント側で弾く
