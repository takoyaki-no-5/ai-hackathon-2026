### 困ったときの順番

### 用語（最初に一度読む）

| 言葉 | 意味 |
|------|------|
| **リポジトリ** | チームのプロジェクト一式。GitHub 上にある。略して「リポ」と言うこともあるが、このノートでは **リポジトリ** と書く |
| **GitHub** | リポジトリを置くサイト。K が F を招待する |
| **コピー（`git clone`）** | リポジトリを初めて自分の Mac に取ってくること（1回だけ） |
| **最新をもらう（`git pull`）** | K が直した最新版を自分の Mac に反映すること。作業のたびに最初にやる |
| **変更を送る（`git push`）** | 自分の変更を GitHub に上げること。**F は基本しない**（K がする） |
| **開発サーバー（`pnpm dev`）** | 自分の Mac 上だけでアプリを一時的に動かす命令 |
| **localhost** | 「このパソコンの中だけ」で見れるアドレス。例: `http://localhost:3000` |
| **公開 URL** | インターネット上の住所。誰でもブラウザで開ける（K が Vercel で作る） |
| **APIキー** | AI など外部サービスを使うための秘密の文字列。**見せない・貼らない** |
| **`.env`** | APIキーなどを書く設定ファイル。**開かない・触らない** |
| **リハーサル（R1 / R2 / R3）** | 本番の前の練習。R2 がいちばん長い（木曜・3時間） |

### 毎回の起動手順

```bash
cd ~/Documents/プロジェクト名   # すでに Cursor で開いていれば、ターミナルはその中にいることが多い
git pull                        # K の最新をもらう
pnpm install                    # 初回だけ、または K が「パッケージ変わった」と言ったとき
pnpm dev                        # 開発サーバーを起動
```

成功の目安: ターミナルに `localhost:3000` などと表示される → Chrome でそのアドレスを開く → 画面が出る。

うまくいかないとき: ターミナルの **赤い文字を全部コピー** → **Cursor**（`Cmd+L`）に貼る。2回試しても直らなければ K。自分で `.env` や Vercel は触らない。
