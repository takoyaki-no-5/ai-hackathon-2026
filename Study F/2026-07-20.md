### 2. プロジェクトをもらって起動（30〜45分）

#### 2-1. GitHub の招待を受ける

- [ ] K から GitHub の招待メールが届く → **Accept**（参加する）を押す（**初回だけ K に頼む**）
- [ ] プロジェクトの URL（`https://github.com/...`）をメモする。メールに無ければ K に聞く

#### 2-2. プロジェクトを Mac にコピーする（初回だけ）

**コピー** = リポジトリを自分の Mac に取ってくること。命令は `git clone`。

```bash
cd ~/Documents
git clone <KからもらったGitHubのURL>
cd <プロジェクトのフォルダ名>
```

`< >` は実際の文字に置き換える。例: `git clone https://github.com/team/my-app.git`

#### 2-3. Cursor でプロジェクトを開く

- [ ] Cursor → **File** → **Open Folder**
- [ ] さっき `cd` した **プロジェクトのフォルダ** を選ぶ
- [ ] 左にファイル一覧が出れば成功

#### 2-4. 必要な部品を入れて、開発サーバーを起動する

```bash
pnpm install
pnpm dev
```

- [ ] ターミナルに `Ready` や `localhost:3000` と出る
- [ ] Chrome のアドレス欄に `http://localhost:3000` と入力して Enter
- [ ] 画面が表示されれば **成功**

**うまくいかないとき**
- ターミナルの **赤いエラー文を全部コピー** → **Cursor**（`Cmd+L`）に貼る。「このエラーを直す手順を教えて」と書く
- Cursor の提案をそのまま試す。 **2回試しても直らない** → K に同じエラー文を送る（スクショでも可）
- `secrets` フォルダや `.env` ファイルの **中身は開かない・チャットに貼らない**（APIキーが入っていることがある）

#### 2-5. 公開 URL（K が作ったら）

**公開 URL** = インターネット上の住所。**作るのは K**（Vercel）。F は届いた URL を開くだけ。

- [ ] K から `https://....vercel.app` のような URL が届いたら、Mac とスマホの Chrome で開く
- [ ] まだ無ければ、リハーサル1 は **localhost（自分の Mac だけ）** で問題ない

#### 2-6. Git の最低限（覚えることだけ）

| 命令 | 意味 | F が使う？ |
|------|------|------------|
| `git pull` | K の最新の変更を自分の Mac に取り込む | **毎回、最初に使う** |
| `git push` | 自分の変更を GitHub に送る | **今日はしない**（基本 K がする） |
