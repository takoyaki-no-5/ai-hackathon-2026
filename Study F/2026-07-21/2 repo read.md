### 2. リポを1ファイルずつ読む

枠: **20〜30分**。**まだコードは変えない**。左の一覧から **1個開く → 読む → 次**。

### 完了の定義

- [ ] 下の **読む順** を上から全部開いた（最後まで行けば OK。全部理解しなくてよい）
- [ ] フォルダの意味を **3つ** 言える（`Study F` / `Concepts` / `secrets` など）
- [ ] 「F が触る／触らない」を Cursor に聞いて答えが返ってきた

### フォルダの地図（先にざっと見る）

| 場所 | F が覚えること |
|------|----------------|
| `Study F/` | **F の日次タスク**（ここを毎日見る） |
| `Study K/` | K 用。F は **見るだけ** でよい |
| `Concepts/` | 用語メモ（短い） |
| `Guides/` | 長めの知見。今日は **README だけ** でよい |
| `secrets/`・`.env` | **開かない・触らない** |
| `app/` | 画面のコード（次のタスクで見る） |

### 読む順（1個ずつ）

Cursor 左の一覧で開く。**チェックを付けながら進む**。

- [ ] `README.md` — リポ全体の索引
- [ ] `overview.md` — 大会のルール・当日
- [ ] `team.md` — K と F の役割
- [ ] `stack.md` — 使う技術の決定（覚えなくてよい。触らない方針だけ）
- [ ] `Study F/README.md` — F の毎日の進め方
- [ ] `study schedule.md` — 勝ち方の型（難しければ流し読み）
- [ ] `tasks.md` — 未決のこと一覧（見るだけ）
- [ ] `Concepts/README.md` — 用語メモの入口
- [ ] `Concepts/vercel.md` — 公開 URL の話
- [ ] `Concepts/zod.md` — 変な入力を防ぐ話

`Guides/` は今日は **開かなくてよい**（後で K が必要なとき案内する）。

### 読み方

1. ファイルを **1つ** 開く
2. 見出しと箇条書きだけ読む（全部暗記しない）
3. わからない **語1つ** だけメモ
4. 次のファイルへ

### Cursor に任せる

全部読んだあと、または途中で `Cmd+L`:

```
このリポで、F（初心者・見せる枝）が
- 毎日見るファイル
- 触っていい場所
- 絶対触らない場所
を3行ずつ教えて。ファイル名も書いて。
```

語の意味だけ聞くとき:

```
「＜わからなかった語＞」はこのプロジェクトで何を指しますか？1行で。
```

### 触らない

- `secrets/`・`.env` の中身
- `.cursor/` の編集
- ファイルの削除・リネーム
