### R2a 仮テーマ読み替え＋尖り選択

枠: **19:45–20:00**（15分・タイマー必須）。担当: **K 決定権、F 提案**。

本番: 当日テーマ受け取り後 **0–15分** と同じ。

### 完了の定義

- [ ] 仮テーマを貼った
- [ ] 読み替え案 **5つ** 出た（発想モード）
- [ ] **3分以内** に1案＋尖り（入力 or 生成）を決定
- [ ] 決定内容を1行メモ（チャット or 紙）

### 手順

#### 19:45 開始（タイマー 15分）

1. F と公開 URL / localhost が開いているか確認
2. 仮テーマを画面に表示（R1 と別の文面推奨）

#### 19:45–19:52 ブレスト（7分）

Cursor または API、`temperature: 1.0`:

```
ハッカソンテーマ: 「＜仮テーマ＞」
字面どおりのSaaSは禁止。読み替え案を5つ、番号付き。
各案: タイトル1行 + 審査員が驚く尖り1行。
```

F の口頭案も歓迎。K が Cursor 案とマージしてよい。

#### 19:52–19:55 決定（3分）

- 1案に絞る（揉めたら **K が切る**）
- 尖り: **画像入力 or 画像生成** のどちらか1つ（水曜までに両方通してあるので、当日気分で選ぶ練習）

メモ例:

```
採用: 「孤独を"一人席のラーメン店"に読み替え」尖り=画像入力で店の雰囲気診断
```

#### 19:55–20:00 共有

F に「何を作るか」を30秒で説明。以降の実装は K。

### やらないこと

- この15分でコードを書き始めない
- 尖りを2つ同時に選ばない
