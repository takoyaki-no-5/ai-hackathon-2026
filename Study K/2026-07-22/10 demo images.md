### #10 デモ用画像2–3枚

枠: **21:45–22:15**（#9 と並行）。担当: **K**（F は場所を覚える）。所要 **10–15分**。

ゴール: 審査・リハで使う **安定した入力画像** がリポにあり、URL から辿れる。

### 完了の定義

- [ ] デモ用画像 **2–3枚** がリポ内にコミットされている
- [ ] 画像入力（#7）で **毎回そこそこ良い結果** が出る
- [ ] F が保存場所を言える

### 手順

#### 1. フォルダ作成

```
public/demo/
  theme-a.jpg
  theme-b.jpg
  theme-c.jpg   # 任意
```

`public/` 配下なら Next から `/demo/theme-a.jpg` で参照可。

#### 2. 画像の選び方

| 良い | 悪い |
|------|------|
| はっきりした被写体・文字少なめ | 真っ白・真っ黒・極端に小さい |
| テーマ読み替えの「尖り」が伝わる | 著作権・顔が特定できる実写（可能なら避ける） |
| 同じ画像で2回デモしても破綻しない | 毎回ランダムすぎる生成物だけに頼る |

自撮り・フリー素材・生成した画像でも可。**2–3枚で固定**。

#### 3. 画面にショートカット（任意・5分）

デモ用ボタン「サンプル画像Aを使う」→ file input に preset URL を渡す、など。無くても **ファイル選択で同じ画像を選ぶ** で可。

#### 4. コミット

```powershell
git add public/demo/
git commit -m "デモ用画像を追加する。"
git push
```

F に「`public/demo/` にある」と伝える。
